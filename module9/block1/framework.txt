 django, flask, bottle и pyramid

 Django: самый мощный
Django — это фреймворк для полноценной full-stack разработки, исторически созданный для задач новостных порталов. В нем есть всё, чтобы создать сервис с фокусом на чтение больших материалов (онлайн-журнал или коллекции блогов), на нём можно писать веб-ресурсы со взаимодействием между пользователями. Django подойдёт вам, если хотите быстро создать прототип-портал с понятным кодом, при этом готовы терпеть навязанные опции.

Flask: гибкость и скромность


Flask — микрофреймворк, «старший брат bottle». Подход к разработке с помощью этого фреймворка можно описать как «ничего лишнего и полная свобода». Здесь нет практически ничего для работы с базами данных, нет шаблонизатора по умолчанию или доступа к базам, но есть варианты, как скомбинировать сборку из нескольких плагинов под решаемую задачу. Из-за высокой вариативности для работы с Flask нужно хорошо разбираться с Python и web.

Pyramid: универсальный фреймворк


Pyramid — web-фреймворк для full-stack разработки, в комплект которого входит много продуманных элементов и решений, среднее между Flask и Django. Он не навязывает жёстких принципов разработки, не настаивает на том, как и где располагается код, позволяет не тратить время на интеграцию сторонних библиотек и не думать о структуре проекта. Можно использовать для быстрого создания прототипа, если не уверены в конечном результате, и хобби-проектов.

Bottle: простота и минимализм


Bottle — пример полноценного микро-фреймворка, который ничего не навязывает, дает тонкую прослойку между пользователем и приложением и идеален для написания API для приложений. Подходит для небольших проектов, где от бэкенда требуется немного действий, нет необходимости в сложных шаблонах или требуется универсальная прослойка для базового контроля. Например, создание с помощью raspberry pi сервиса по управлению светом в коридоре с возможностью проверять состояние через веб, написание чат-ботов или сервисов, использующих их как реакцию на внешнее действие.
